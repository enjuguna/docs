## General Usage

### How to Launch the Application (Backend)

**Pre-Deployment Checklist**
- Ensure `inverter_server.exe` exists in the `production/` folder.
  - If missing, build it:
    ```
    cd api
    go build -o ../production/inverter_server.exe
    ```
- Verify all CSV folders exist (`inverters/`, `models/`, `results/`, `logs/`, `tests/`). The API will auto-create missing files.

**Starting the Backend Server**
- Run from project root or production folder:
    ```
    cd production
    start.bat
    ```
- This launches the server on port 3031.

To confirm:
- Open browser: `http://localhost:3031/api/v1/health`
- Or run:
    ```
    curl http://localhost:3031/api/v1/health
    ```

**Stopping the Backend Server**
- From the same folder, run:
    ```
    stop.bat
    ```
- Manual method:
    ```
    netstat -ano | findstr :3031
    taskkill /PID <PID> /F
    ```

### Launching the Application (Frontend Web/Electron)

**Web Version**
- From `frontend` folder:
    ```
    npm install       // install dependencies
    npm run dev       // start Vite dev server
    ```
- Open browser at [`http://localhost:8080`](http://localhost:8080).

**Electron Desktop Version**
- From `frontend` folder:
    ```
    npm install
    npm run electron
    ```
- Opens app in desktop window (if Electron set up).

**Expiration Lock**
- App locks after September 30, 2025 (see [`ExpirationContext.tsx`](frontend/src/contexts/ExpirationContext.tsx:1)), blocking UI.
- Change `EXPIRY_UTC` for testing.

### Login & Authentication

- Login screen requires username/password (handled by [`AuthContext.tsx`](frontend/src/contexts/AuthContext.tsx:1)).
- Session stored locally.
- All tabs protected (redirect to login until authenticated).
- No backend authentication enabled by default (future enhancement can add JWT/API key).

---

### Navigating the Interface

- Tabs: Inverters, Models, Tests, Verification, Logs, Remote Sites.
- Click a tab to switch to its section.
- Menu bar offers Help and About options.

### Data Storage

- All dynamic data persists in CSV files for easy backup and integration.
    - Inverter data: `inverters/inverters.csv`
    - Model data: `models/`
    - Results: `results/`
    - Logs: `logs/application.log`
    - Tests: `tests/`
