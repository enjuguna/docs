## Troubleshooting & Error Handling

### Common API & Application Errors

- **API returns HTTP codes:**
    - 200/201 OK/Created: Success
    - 400 Bad Request: Validation error, check required fields
    - 404 Not Found: CSV/model/inverter missing
    - 500 Server Error: General system issue

- **App does not start:**
  - Ensure all dependencies (Windows 11, Go backend, Node/electron for frontend) are installed.
  - Confirm presence of `inverter_server.exe`, required folders (`inverters/`, `models/`, etc.).
  - Check for missing files, review error logs at [`logs/application.log`](logs/application.log:1).

- **Verification/test fails:**
  - Make sure models and registers are valid.
  - Confirm inverters exist and Modbus port (`502`) is accessible.
  - Review front and backend logs for `[ERROR]` entries.

- **Register test shows "N/A":**
  Device may be unreachable or register info invalid.

- **Logs not updating:**
  - Ensure write permission to [`logs/application.log`](logs/application.log:1).

  - Change `EXPIRY_UTC` to a future date for continued testing.

### Tips

- Regularly export and back up your CSV files.
- Use descriptive names for inverters and models.
- Review API docs and Help menu for further resources.
- Regenerate `inverter_server.exe` after backend updates for changes to take effect.
